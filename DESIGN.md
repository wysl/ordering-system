# 点餐系统设计文档 v3

## 需求变更

### 1. 人员档案管理
- 管理后台新增 CSV 导入人员名单
- CSV 格式：`姓名` (单列)
- 新增 Person 表存储人员名单
- 选餐页面：姓名从输入框改为下拉选择（读取人员名单）

### 2. 单选逻辑
- 每个人只能选一道菜品 + 对应辣度
- 选中一道后，其他菜品变灰不可选
- 再次点击已选中的菜品 → 解锁（取消选择），可以选其他
- 数量固定为 1

### 3. 订单修改支持
- 同一人重新提交订单时，覆盖之前的订单（删除旧 order，创建新 order）
- 选餐页面：选择姓名后，自动加载该人已提交的订单（高亮显示已选菜品）
- 再次点击已选菜品可解锁，重新选择提交

## 数据库新增

### Person 人员表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | uint | 主键 |
| name | string | 姓名（唯一） |

## API 变更

### 新增
- `POST /api/admin/personnel/import` — CSV 导入人员名单（覆盖模式）
- `GET /api/personnel` — 获取人员列表（公开）
- `GET /api/order/mine?person=xxx` — 查询某人已提交的订单

### 修改
- `POST /api/order` — 如果该人已有订单，先删除旧订单再创建新订单
